VẤN ĐỀ CODE CŨ (cần fix khi rebuild)
1. N+1 LookUp — gọi SharePoint 7 lần cùng 1 Username:
// Cũ: 7 lần LookUp
Set(userrolevalue, LookUp(USER, Username = ...).Role)
Set(usern, LookUp(USER, Username = ...).Fullname)
// ...
// Mới: 1 lần, dùng With()
With(
    {_u: LookUp(USER, Username = txtUsername.Text And Password = txtPassword.Text)},
    If(IsBlankOrError(_u),
        Notify("Sai tài khoản hoặc mật khẩu"),
        Set(userrolevalue, _u.Role);
        Set(usern, _u.Fullname);
        Set(permission, _u.Permission);
        Set(userid, _u.ID);
        Set(userimg, _u.'Hình ảnh');
        Set(usernh, _u.'Ngành hàng');
        Set(user, _u.Username);
        Navigate('Trang Chủ', ScreenTransition.Cover)
    )
)
2. Switch 37 cases — tất cả đều Navigate cùng 1 màn hình
 bỏ hoàn toàn, dùng Navigate thẳng sau khi xác thực.
3. Password plain text
 Cần cân nhắc dùng Azure AD / Entra ID thay vì tự quản lý bảng USER.
4. Show/hide password dùng
 pattern này hoạt động nhưng hacky, rebuild nên dùng toggle variable.

Vấn đề cần lưu ý khi rebuild
If-chain 19 nhánh — Anti-pattern nghiêm trọng. Khi rebuild nên dùng 1 SharePoint config table mapping userrolevalue → Quầy hàng, sau đó LookUp ra giá trị filter. Giảm từ 19 nhánh → 1 dòng Filter.
Date picker đặt tên ngược — dpStartDate_2 có placeholder "Đến ngày" và ngược lại. Logic filter cũng bị ngược tương ứng. Cần kiểm tra lại khi rebuild.
Softline/Hardline dùng sai date pickers — tham chiếu dpEndDate/dpStartDate (không có _2) không tồn tại trong screen → filter ngày không hoạt động cho 2 role này.
AddColumns không cần thiết — chỉ để alias ID thành sortID. Xóa đi, sort trực tiếp trên ID.
Delegation risk — Filter với Or() lồng nhau trên date + Ngành hàng có thể không delegate được nếu list > 500/2000 records.

Vấn đề cần lưu ý khi rebuild
Không set varItem sau Patch — sau khi tạo record xong, navigate về list nhưng không set varItem = record vừa tạo. User phải tìm và click record mới trong list để tiếp tục → UX kém. Nên dùng Set(varItem, Patch(...)) để capture kết quả, rồi navigate thẳng vào detail screen.
Không có error handling — Patch() không có IfError(). Nếu ghi thất bại, user không biết. Nên thêm:
powerfx
   If(IsError(Patch(...)), Notify("Lỗi lưu dữ liệu", NotificationType.Error))
Không validate — có thể submit với dropdown ở giá trị mặc định (blank/empty). Nên disable btn_insert_1 khi dd_department.Selected.Value = Blank().
Distinct() trên cascading dropdown — nếu THÔNG TIN QUẦY HÀNG VMD lớn, Distinct không delegate. Với bảng config nhỏ thì ổn, nhưng cần document rõ giới hạn này.
Bảng THÔNG TIN QUẦY HÀNG VMD — đây là bảng master data quan trọng. Khi rebuild cần đảm bảo dữ liệu đồng bộ với logic filter ở screen 'Chấm điểm VMD' (19 nhánh If hardcode tên Quầy hàng).

  → AddColumns: 'Số lần lỗi' = CountRows('Nhóm quầy hàng')
  → AddColumns: Nhãn = "Quầy hàng (Số lần lỗi)"
Output: X-axis = Nhãn (Tên quầy + count), Y-axis = Số lần lỗi
