LỚP 3: QUYỀN GIỚI HẠN DỮ LIỆU Row-Level Security
Câu hỏi: Bạn được nhìn thấy dữ liệu của những ai?

Bảo mật cấp dòng (RLS) lọc dữ liệu hiển thị theo Quầy hàng dựa trên 37 Roles cứng.

Chi Tiết Lớp 1: Quyền truy cập Module

Quyết định chức năng nào được hiển thị trên thanh điều hướng hoặc màn hình điều khiển chính dựa vào thuộc tính Ngành hàng NhomNganhHang.

Cơ cấu dữ liệu & Bảng mapping
Nhóm ngành hàng Module được phép thấy
FOODLINE 1, FOODLINE 2 Chấm điểm POP, Mượn trả vật tư, Báo cáo POP
SOFTLINE, HARDLINE Chấm điểm POP, Mượn trả vật tư, Báo cáo, Chấm điểm VMD, Kiểm tra VMD
SPA, SM, Test Toàn bộ các module ứng dụng + Cấu hình Admin hệ thống
Mô phỏng Code lọc Menu (React / JS)
// Lọc danh sách menu dựa trên trường NhomNganhHang của User
const navigationItems = [
  { id: 'pop_score', label: 'Chấm Điểm POP', allowed: ['Food', 'Softline', 'Hardline', 'SPA'] },
  { id: 'vmd_score', label: 'Chấm Điểm VMD', allowed: ['Softline', 'Hardline', 'SPA'] },
  { id: 'configs', label: 'Cấu hình hệ thống', allowed: ['SPA'] }
];

const allowedItems = navigationItems.filter(item => 
  item.allowed.includes(currentUser.department) || currentUser.permission === 'SPA'
);

Chi Tiết Lớp 2: Quyền thao tác dữ liệu (CRUD)

Quyết định thao tác nghiệp vụ được phép thực hiện trên dữ liệu hiển thị (Thêm mới bản ghi chấm điểm, Sửa lỗi, Phản hồi lỗi, Xóa bản ghi lỗi).

Phân tích ma trận quyền
Quyền (Permission) Xem Thêm mới Khắc phục lỗi Xóa
SPA (Admin) Có Có Có Có
Ngành hàng (GL) Có Không Có (Upload ảnh sửa) Không
Test (Tester) Có Không (Khóa tránh rác) Không Không
Mô phỏng logic UI khóa nút bấm (React)

Chi Tiết Lớp 3: Quyền giới hạn dữ liệu (Row-Level Security)

Cơ chế tối ưu hóa bảo mật cấp dòng bằng **Bảng Từ Điển (Lookup Table)**. Tránh việc dùng hơn 35 câu lệnh If rườm rà.

Mô hình hoạt động của database mới

Khi đăng nhập, hệ thống tra cứu bảng THONG_TIN_QUAY_HANG để biết vai trò hiện tại ứng với các quầy hàng nào, sau đó dùng giá trị đó để lọc động dữ liệu hiển thị.

Ví dụ so sánh mã nguồn phân luồng
Phiên bản Cách xử lý phân luồng
Hệ thống cũ
(Anti-pattern)		 
If(Role = "GL Ladies", Filter(POP, Quay = "Ladies"),
   Role = "GL Men", Filter(POP, Quay = "Men"),
   Role = "GL Beverge", Filter(POP, Quay = "Beverge"),
   ... 35 dòng If lồng nhau ...)
Database mới
(Lookup Dictionary)		 
-- Lấy danh sách quầy được lọc từ từ điển
Set(gblUserAllowedQuays, 
    Filter(THONG_TIN_QUAY_HANG, VaiTroMacDinh = CurrentUser.VaiTro)
);
-- Lọc Gallery POP tự động, không cần If
Filter(CHAM_DIEM_POP, QuayHang in gblUserAllowedQuays)
Câu lệnh SQL tối ưu cho Row-Level Security
SELECT p.* 
FROM CHAM_DIEM_POP p
LEFT JOIN THONG_TIN_QUAY_HANG q ON p.QuayHang = q.QuayHang
WHERE 
    q.VaiTroMacDinh = :currentUserRole
    OR :currentUserPermission IN ('SPA', 'Test', 'SM');
